# Task Decomposition & Subagent Orchestration Skill

This skill details how the main agent decomposes complex requirements into self-contained sub-tasks, coordinates subagent execution, and aggregates their contributions.

## When to Use
- Approaching a large, multi-step feature implementation.
- Handling tasks that can be parallelized or researched independently.
- Coordinating background subagents to solve separate sub-tasks.

## Guidelines & Rules

### 1. Decomposition & Progress Checking
- Break large user requirements down into a sequence of smaller, independent sub-tasks.
- Maintain a local task checklist (e.g., `progress.md` or a tracking artifact) at the project root to map completed and outstanding tasks.

### 2. Subagent Provisioning
- Define specialized subagents using precise instructions.
- Give each subagent a highly focused, narrow, and non-overlapping scope to prevent redundant work.
- Provide subagents with only the files and lines of context they need, keeping their context windows clean and context-efficient.

### 3. Synthesis & Merge
- Systematically merge the code, tests, and documentation generated by subagents.
- Ensure interface integrity: check that the new components integrate properly.
- Run the full suite of integration tests to verify the unified system before finalizing the implementation.
